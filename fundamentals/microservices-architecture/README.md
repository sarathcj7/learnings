# Microservices Architecture

Building systems with small, independent services. Domain-driven design, service boundaries, distributed communication.

## Why This Matters

- **Scale**: Each service scales independently
- **Deployability**: Deploy one service without affecting others
- **Team autonomy**: Teams own services end-to-end
- **Complexity**: Distributed systems challenges (testing, debugging, consistency)
- **Interview**: "Design for 1000 microservices" scenarios
